# 사용자 시나리오

## 개요

| 항목 | 내용 |
|------|------|
| 서비스 | N배송 (네이버 플러스 스토어 내 배송 서비스) |
| 대상 화면 | HOME_001, PRODUCT_001, SHOP_001, PAY_001 |
| 작성일 | 2026/05/09 |

---

## 시나리오 1. 개인화 추천 상품 탐색 후 구매 완료 (메인 플로우)

> 사용자가 N배송 홈에 진입해 개인화 추천 상품을 발견하고, 쿠폰을 수령한 뒤 결제까지 완료하는 핵심 구매 여정

### 전제 조건
- 사용자는 네이버에 로그인된 상태
- N Pay 포인트 보유 중
- 배송지가 기 설정되어 있음

### 플로우

**[홈 진입]**
1. 사용자가 네이버 플러스 스토어에서 N배송 홈(HOME_001)으로 진입한다.
2. 상단 GNB에서 현재 배송지를 확인한다.
3. '내일배송 개인화 추천' 섹션의 소제목을 통해 자신의 관심사 기반 추천임을 인지한다.
4. 가로 스크롤로 추천 상품 목록을 훑어보며 관심 상품을 탐색한다.
   - 각 카드에 표시된 '내일배송' 뱃지를 통해 익일 도착 가능 여부를 확인한다.
5. 관심 상품 카드를 탭하여 상품 상세(PRODUCT_001)로 이동한다.

**[상품 상세 확인]**
6. 상품 이미지 슬라이드를 좌우로 넘기며 상품을 확인한다.
   - 멤버십 배지를 통해 무료 교환/반품 혜택을 인지한다.
7. 상품 정보 영역에서 정가 → 할인가 → 나의 할인가 순으로 최종 가격을 확인한다.
8. 별점/리뷰 수를 탭하여 리뷰 영역으로 스크롤 이동해 후기를 확인한다.
9. 배송 영역에서 '내일배송' 뱃지와 도착 예정일을 확인한다.
10. 주문 마감 카운트다운 타이머를 확인해 오늘 주문 시 내일 도착 가능 여부를 판단한다.
11. 쿠폰받기 버튼(D-day 표시)을 탭하여 쿠폰을 다운로드한다.
12. 적립 영역에서 이번 구매로 적립될 멤버십 포인트를 확인한다.
13. 혜택 영역에서 카드 추가 적립, 무이자 할부 혜택을 확인한다.
14. 장바구니 버튼을 탭하여 상품을 장바구니에 담는다.
    - GNB 장바구니 버튼의 뱃지 숫자가 증가함을 확인한다.

**[장바구니 검토]**
15. GNB 장바구니 버튼을 탭하여 장바구니(SHOP_001)로 이동한다.
16. '일반배송' 탭이 선택된 상태에서 담은 상품을 확인한다.
17. 배송지 영역에서 배송받을 주소를 재확인한다.
18. 개별 상품 카드에서 스토어명, 도착 예정일, 가격을 최종 확인한다.
19. 카운트다운 타이머를 보며 내일배송 마감 시간을 재확인한다.
20. 상품 개수 조정 버튼(+)을 눌러 수량을 2개로 변경한다.
21. N쿠폰받기 버튼을 탭하여 해당 스토어의 추가 쿠폰을 확인한다.
22. 주문 예상 금액 요약 영역에서 총 금액, 즉시할인, 쿠폰할인, 배송비를 확인한다.
23. 전체선택 체크박스로 구매할 상품을 선택한 뒤 결제하기 버튼을 탭한다.

**[결제 진행]**
24. 결제 화면(PAY_001)으로 진입한다.
25. N Pay 포인트/머니 영역에서 현재 보유 포인트를 확인한다.
26. 포인트 사용 입력 필드에 사용할 금액을 직접 입력한다.
    - 입력값이 녹색으로 표시됨을 확인한다.
27. 결제 적립 포인트 안내 영역에서 이번 구매로 적립될 포인트를 확인한다.
28. 개인정보 제공동의 영역을 확인한 뒤 '충전 후 결제하기' 버튼을 탭하여 결제를 완료한다.

## 시나리오 2. 타임딜 상품 긴박 구매

> 홈에서 타임딜 섹션을 발견하고 마감 전에 빠르게 구매하는 시나리오

### 플로우

1. 홈 스크롤 중 타임딜 섹션의 카운트다운 타이머(HH:MM:SS)를 발견한다.
2. 남은 시간이 얼마 없음을 인지하고 프로모션 태그(예: 브랜드데이)가 붙은 상품을 탭한다.
3. 상품 상세(PRODUCT_001)에서 주문 마감 카운트다운 타이머가 빨간색으로 강조된 것을 확인한다.
   - 마감이 임박한 상태임을 시각적으로 인지한다.
4. 가격 및 할인 정보를 빠르게 확인하고 장바구니에 담는다.
5. 즉시 장바구니(SHOP_001)로 이동하여 상품을 선택한다.
6. 카운트다운 타이머가 빨간색으로 강조된 것을 재확인하며 결제하기를 탭한다.
7. 결제 화면(PAY_001)에서 포인트를 전액 사용 버튼으로 적용하고 결제를 완료한다.

---

## 시나리오 3. 당일배송 상품 급구매

> 당일 수령이 필요한 사용자가 마감 전 빠르게 주문하는 시나리오

### 플로우

1. 홈에서 당일배송 안내 배너를 통해 마감 시각을 확인한다.
2. 배너의 '>' 버튼을 탭하여 당일배송 상품 목록으로 이동한다.
3. 원하는 상품을 탭하여 상품 상세(PRODUCT_001)로 진입한다.
4. 배송 영역에서 도착 예정일(당일)과 카운트다운 타이머를 확인한다.
   - 타이머가 빨간색으로 강조되어 마감 임박을 경고한다.
5. 즉시 장바구니에 담고, 장바구니에서 바로 결제로 진행한다.
6. 결제 화면에서 포인트 설정 없이 '충전 후 결제하기'를 탭하여 결제를 완료한다.

---

## 시나리오 4. 카테고리 탐색 후 구매

> 원하는 카테고리를 직접 골라 상품을 탐색하는 시나리오

### 플로우

1. 홈에서 카테고리 영역을 확인한다. 기본적으로 맞춤형 상위 10개만 노출된다.
2. 원하는 카테고리가 보이지 않아 '전체 카테고리 더보기'를 탭한다.
   - 25개 전체 카테고리가 펼쳐진다.
3. 원하는 카테고리를 탭하여 해당 카테고리 상품 목록으로 이동한다.
4. 이후 '^' 버튼으로 카테고리를 다시 접는다.
5. 상품을 선택하여 상세 → 장바구니 → 결제 순으로 구매를 완료한다.

---

## 시나리오 5. 장바구니에서 상품 수정 후 구매

> 여러 상품을 담아두고 장바구니에서 수량·옵션 조정 후 구매하는 시나리오

### 플로우

**[상품 담기]**
1. 홈에서 여러 상품을 탐색하며 각 상품 상세(PRODUCT_001)에서 장바구니에 차례로 담는다.
2. GNB 장바구니 버튼 뱃지에서 누적된 상품 수를 확인한다.

**[장바구니 수정]**
3. 장바구니(SHOP_001)에 진입하여 담은 상품 목록을 확인한다.
4. 상품 A의 수량을 3개로 올리기 위해 '+' 버튼을 두 번 탭한다.
5. 상품 B의 옵션을 변경하기 위해 '옵션 변경' 버튼을 탭한다.
   - 하단에서 옵션 변경 모달이 올라온다.
   - 원하는 옵션을 선택하고 '적용'을 탭한다.
   - 상품 B의 정보가 변경된 옵션으로 업데이트된다.
6. 상품 C는 구매하지 않기로 하여 'X' 삭제 버튼을 탭한다.
   - 상품 C가 즉시 장바구니에서 사라진다.
7. 상품 D의 수량을 '-' 버튼으로 줄이려다 1개에서 한 번 더 누른다.
   - '1개 이하는 구입할 수 없습니다' 알림창이 노출된다.
   - 확인을 탭하고 수량을 유지한다.

**[금액 확인 및 결제]**
8. 전체선택 체크박스를 탭하여 남은 상품을 모두 선택한다.
9. 주문 예상 금액 요약 영역에서 총 금액 및 할인 내역을 최종 확인한다.
10. 결제하기를 탭하여 결제 화면(PAY_001)으로 이동하고 결제를 완료한다.

---

## 시나리오 6. N Pay 포인트 전액 사용 결제

> 보유한 포인트를 전부 소진하여 결제하는 시나리오

### 플로우

1. 결제 화면(PAY_001)에 진입한다.
2. N Pay 포인트/머니 영역에서 '?' 도움말 버튼을 탭하여 포인트와 머니의 차이를 확인한다.
3. '내 숨은 포인트 찾기' 버튼을 탭하여 추가 수령 가능한 포인트를 확인한다.
4. '전액 사용' 버튼을 탭한다.
   - 보유 포인트 전액이 포인트 사용 입력 필드에 자동 입력된다.
   - 입력값이 녹색으로 표시된다.
5. 이후에도 매 구매마다 자동으로 전액 사용하도록 '항상 전액사용' 버튼을 활성화한다.
6. 결제 적립 안내에서 '네이버페이 머니 결제적립' 기준 포인트가 표시됨을 확인한다.
7. '충전 후 결제하기' 버튼을 탭하여 결제를 완료한다.

---

## 시나리오 7. 포인트 초과 입력 오류 처리

> 보유 포인트보다 많은 금액을 입력했을 때의 예외 처리 시나리오

### 플로우

1. 결제 화면(PAY_001)에서 포인트 사용 입력 필드를 탭한다.
2. 보유 포인트보다 큰 금액을 직접 입력한다.
3. '보유 금액 이상 사용은 불가능 합니다.' 팝업창이 화면에 노출된다.
4. '확인' 버튼을 탭한다.
   - 입력 필드의 숫자가 자동으로 삭제되어 빈 상태로 돌아간다.
5. 올바른 금액을 다시 입력하거나 '전액 사용' 버튼을 사용한다.
6. 결제를 완료한다.

---

## 시나리오 8. 후불 결제 신청 및 적용 (미신청 유저)

> 후불 결제를 처음 신청하고 바로 적용하여 결제하는 시나리오

### 플로우

1. 결제 화면(PAY_001)에서 후불 결제 영역을 발견한다.
2. 후불 결제 좌측 체크 아이콘을 탭한다.
   - 미신청 유저이므로 하단에서 후불 결제 신청 모달창이 올라온다.
3. 모달에서 후불 결제 조건을 확인하고 신청을 완료한다.
4. 후불 결제 적용 UI로 화면이 변경된다.
5. 후불 결제가 선택된 상태로 결제를 진행한다.

---

## 시나리오 9. 결제 취소 후 재진행

> 결제 도중 취소하고 상품을 재검토한 뒤 다시 결제하는 시나리오

### 플로우

1. 결제 화면(PAY_001)에 진입한다.
2. 포인트 설정 도중 상품을 다시 확인하고 싶어 상단의 'X' 취소 버튼을 탭한다.
   - 결제 페이지가 종료되고 이전에 보고 있던 상품 상세(PRODUCT_001) 화면으로 돌아간다.
3. 상품 정보, 혜택, 배송 조건을 재확인한다.
4. 장바구니 버튼을 탭하여 장바구니(SHOP_001)로 이동한다.
5. 상품 선택 후 다시 결제하기를 탭하여 결제 화면(PAY_001)으로 재진입한다.
6. 이번엔 '항상 전액사용' 버튼을 활성화하고 결제를 완료한다.

---

## 시나리오 10. 배송지 변경 후 배송 정보 갱신

> 배송지를 변경하여 달라진 배송 정보와 상품을 탐색하는 시나리오

### 플로우

1. 홈(HOME_001)에서 GNB의 주소 변경 버튼을 탭한다.
2. 배송지를 새 주소로 변경한다.
   - NFA(Naver Fulfillment Alliance) 물류센터와의 거리/입고 여부에 따라 상품 및 배송 정보가 갱신된다.
3. 변경된 배송지 기준으로 내일배송 가능 상품 목록이 업데이트됨을 확인한다.
4. 개인화 추천 섹션을 탐색하며 새 배송지에서 이용 가능한 상품을 확인한다.
5. 상품 상세(PRODUCT_001)에서 변경된 배송지 기준의 도착 예정일을 확인한다.
6. 원하는 상품을 장바구니에 담고 결제를 진행한다.
   - 장바구니(SHOP_001)에서도 변경된 배송지가 배송지 영역에 표시됨을 확인한다.

---

## 시나리오 11. 일부 상품만 선택해 구매 (선택 구매)

> 장바구니에 담긴 여러 상품 중 일부만 선택하여 결제하는 시나리오

### 플로우

1. 장바구니(SHOP_001)에 여러 상품이 담겨 있는 상태로 진입한다.
2. 전체선택 체크박스를 탭하여 전체 선택한다.
3. 이번에 구매하지 않을 상품의 체크박스를 개별 해제한다.
4. 주문 예상 금액 요약 영역에서 선택한 상품 기준의 총 금액이 실시간으로 반영됨을 확인한다.
   - 즉시할인, 쿠폰할인, 배송비도 선택 상품 기준으로 재계산된다.
5. 선택삭제 버튼을 실수로 탭한다.
   - 선택된 상품만 일괄 삭제된다.
   - 삭제 전 상품만 장바구니에 남아 있다.
6. 남은 상품을 확인하고 결제를 진행한다.
